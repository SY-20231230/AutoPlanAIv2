# 파일 역할 정리

이 문서는 현재 폴더에 있는 각 파일과 폴더의 역할을 설명합니다.

## 🤖 1. 아이디어 기획 시스템 (최근 생성)
사용자의 아이디어를 입력받아 대화를 통해 구체적인 기획서로 발전시키는 시스템입니다.

- `idea_to_plan_generator.py`: **(메인 실행 파일)** 전체 아이디어 기획 프로세스를 총괄하고 관리합니다. 시스템을 테스트하려면 이 파일을 실행하세요.
- `idea_refiner.py`: **(아이디어 정제기)** 사용자의 초기 아이디어를 '목표', '타겟' 등 구조화된 데이터로 정리합니다.
- `idea_expander.py`: **(아이디어 확장기)** 정리된 아이디어를 바탕으로 추가할 만한 기능들을 추천합니다.
- `conversation_manager.py`: **(대화 관리자)** 사용자와의 대화를 해석하고, 피드백을 기획 내용에 실시간으로 반영합니다.
- `plan_generator.py`: **(기획서 생성기)** 최종 확정된 아이디어를 바탕으로 마크다운(.md) 형식의 기획서 파일을 생성합니다.

---

## 📑 2. 깃허브 분석 시스템 (기존)
프로젝트 기획서와 유사한 깃허브 오픈소스를 찾아 분석하는 시스템입니다.

- `gemini_refiner.py`: **(Gemini 정제기)** 기획서 원문(JSON)을 입력받아, Gemini AI를 통해 더 상세한 기술 기획서로 재작성합니다.
- `github_num.py`: **(메인 실행 파일)** '내 기획서'와 가장 유사한 GitHub 저장소 Top 5를 Gemini AI로 평가하고 찾아줍니다.
- `github_crawler.py`: **(GitHub 크롤러)** 특정 조건의 GitHub 저장소 정보를 수집합니다.
- `keyword_extractor.py`: **(키워드 추출기)** 텍스트에서 핵심 키워드를 추출합니다.
- `similarity_analyzer.py`: **(유사도 분석기)** 텍스트 또는 데이터 간의 유사도를 계산합니다.
- `task_allocator.py`: **(작업 할당기)** 분석 결과를 바탕으로 개발 작업을 할당하는 기능으로 추정됩니다.

---

## 💾 3. 데이터 및 기타 파일
위 시스템들에서 사용되거나 생성된 데이터 파일 및 기타 파일입니다.

- `데이터/` (폴더): 각종 JSON, CSV 데이터 파일을 모아두는 곳입니다.
- `features_*.json`: 기능 명세, 기획서 원문 등이 포함된 JSON 데이터 파일입니다.
- `github_repositories.json`: `github_crawler.py`가 수집한 저장소 목록 데이터입니다.
- `github_top5_similar_gemini.json`: `github_num.py`가 찾아낸 Top 5 유사 저장소 결과 파일입니다.
- `keywords.json`: 키워드 목록이 저장된 데이터 파일입니다.
- `development_plan_*.csv`: CSV 형식으로 작성된 개발 계획 파일입니다.
- `project_plan_8.csv`: CSV 형식의 프로젝트 계획 파일입니다.
- `analysis_report.md`: 분석 결과에 대한 보고서 파일입니다.
- `20250805백업/`: 중요한 파일을 백업해두는 폴더입니다.
- `test01.py`: 간단한 기능 테스트를 위한 스크립트 파일입니다.
